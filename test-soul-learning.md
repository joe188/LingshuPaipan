# 测试任务：学习 SOUL.md 中的开发规则

## 任务目标
让 MemOS 学习并记住 SOUL.md 中的核心开发规则，以便在未来开发中自动应用。

## SOUL.md 核心规则

### 1. 技术栈原则
- React Native 0.85.1 + Fabric 新架构（不可降级）
- 使用 @op-engineering/op-sqlite（禁用 expo-sqlite）
- 不轻易改动依赖版本

### 2. APK 发布流程（7 步）
1. 完成代码开发
2. 在模拟器上测试所有功能
3. 确认所有功能正常
4. 修复所有问题
5. 再次测试确认
6. 构建正式版 APK
7. 发给用户测试

### 3. 禁止事项
- ❌ 未测试就构建 APK
- ❌ 未测试就发给用户
- ❌ 擅自恢复文件
- ❌ 建议降级 React Native

### 4. 代码质量
- 所有代码必须经过 TypeScript 检查
- 所有功能必须在模拟器测试
- 用户反馈的问题必须记录并修复

## 预期效果
MemOS 应该：
1. 自动记录这些规则到记忆
2. 下次开发时自动提醒
3. 如果违反规则，自动警告
