## React Native 数据库迁移 - 成功！

### 问题
应用在 Android 模拟器上启动时崩溃，SQLite 数据库初始化失败。

### 根本原因
- 最初尝试 `react-native-quick-sqlite@5.0.0`，但该库的 build.gradle 假设 RN <=0.70 的目录结构，对 RN 0.71+（使用 `ReactAndroid` 而非 `android`）不兼容，导致 Gradle 构建失败。
- 回退到 `react-native-sqlite-storage@6.0.1`，但出现 `Cannot convert null value to object` 错误。

### 解决方案（react-native-sqlite-storage）
1. **正确安装和链接**：
   - `npm install react-native-sqlite-storage@6.0.1 --legacy-peer-deps`
   - 手动配置 `settings.gradle` 指向 `platforms/android`
   - 手动配置 `MainApplication.kt` 添加 `SQLitePluginPackage`
   - 手动配置 `app/build.gradle` 添加 `implementation project(':react-native-sqlite-storage')`

2. **简化 Database.ts**：
   - 使用回调风格 `SQLite.openDatabase(config, successCb, errorCb)` 而非 async/await
   - 不指定 `androidDatabaseImplementation` 参数（由库自动选择）
   - 添加详细日志以便调试

### 验证结果
✅ 应用成功安装并启动
✅ 数据库打开成功
✅ 6个表创建成功（history, settings, eight_ga, hexagrams, yao_shi, liuyao_interpretation）
✅ 种子数据完整导入：
   - 8 条八卦
   - 64 条六十四卦
   - 458 条六爻解读

### 配置文件摘要
- `android/settings.gradle`:
  ```gradle
  include ':react-native-sqlite-storage'
  project(':react-native-sqlite-storage').projectDir = new File(rootProject.projectDir, '../node_modules/react-native-sqlite-storage/platforms/android')
  ```
- `android/app/src/main/java/com/lingshu/paipan/MainApplication.kt`:
  ```kotlin
  import org.pgsqlite.SQLitePluginPackage
  // 在getPackages()中添加 SQLitePluginPackage()
  ```

### 当前状态
数据库在 Android 模拟器上运行正常。应用可以启动并初始化 SQLite 数据库。

---

**备注**：`react-native-quick-sqlite` 需要 RN >=0.74 或复杂的 build.gradle patch 才能在 RN 0.72 上工作。使用 `react-native-sqlite-storage` 是简单可靠的方案。

## React Native 数据库迁移和验证 - 2026-04-15

### 问题背景
应用在 Android 模拟器上启动时出现 `TypeError: Cannot convert null value to object` 错误，定位到数据库初始化阶段。

### 根本原因分析
1. 最初尝试使用 `react-native-quick-sqlite@5.0.0`，但该库的 build.gradle 假设 RN <=0.70 的目录结构，对 RN 0.71+（使用 `ReactAndroid` 而非 `android`）不兼容，导致 Gradle 构建失败。
2. 回退到 `react-native-sqlite-storage@6.0.1`，但出现 `Cannot convert null value to object` 错误。
3. 最终迁移到 `expo-sqlite`，但在测试时发现数据库API使用错误：
   - 错误地使用 `execAsync()` 来执行 SELECT 查询（应该使用 `getAllAsync()`）
   - `execAsync()` 只用于执行不返回结果的SQL（INSERT/UPDATE/DELETE）

### 解决方案实施
1. **正确配置 Gradle**：
   - AGP 版本 7.3.1 与 Kotlin 1.7.22 对齐
   - Repository 顺序：mavenCentral 在 google() 之前以避免网络超时
   - 添加 `reactNativeVersion = "+"` 到 root build.gradle

2. **数据库API修正**：
   - 修改 `executeSql` 方法使用正确的API：
     ```typescript
     async executeSql<T = any>(
       sql: string,
       params: any[] = []
     ): DbPromise<{ rows: { item: (index: number) => T; length: number } }> {
       if (!this.db) throw new Error('Database not initialized');
       try {
         // 错误修正：SELECT 查询应使用 getAllAsync 而不是 execAsync
         if (sql.trim().toUpperCase().startsWith('SELECT')) {
           const result = await this.db.getAllAsync(sql, params);
           return {
             rows: {
               get length() {
                 return result.length;
               },
               item: (index: number) => result[index] as T,
             },
           };
         } else {
           // 非SELECT语句使用 execAsync
           await this.db.execAsync(sql, params);
           return {
             rows: {
               get length() {
                 return 0;
               },
               item: (index: number) => undefined as T,
             },
           };
         }
       } catch (error) {
         console.error('SQL Error:', sql, error);
         throw error;
       }
     }
     ```

3. **测试验证**：
   - APK 成功生成
   - 模拟器连接 Metro 成功
   - 应用启动时数据库初始化正常
