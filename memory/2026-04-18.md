## 灵枢排盘应用 - AI解析功能修复

### 问题背景
用户使用本地大模型软件MNN chat测试灵枢排盘应用的AI解析功能，但是遇到以下问题：
1. 测试连接失败，错误提示"Network request failed"
2. 无法获取模型列表
3. AI解析功能一直显示"AI正在分析中"，无法获取数据

### 解决过程

#### 1. 识别根本原因
- **Release版本的APK禁止了HTTP明文流量**（`usesCleartextTraffic="false"`）
- 这是因为React Native的默认配置中，Release版本禁止HTTP明文流量
- Debug版本允许HTTP明文流量（`usesCleartextTraffic="true"`）

#### 2. 修复步骤

##### 2.1 添加网络安全配置
创建了`android/app/src/main/res/xml/network_security_config.xml`：
```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <domain-config cleartextTrafficPermitted="true">
        <domain includeSubdomains="true">localhost</domain>
        <domain includeSubdomains="true">127.0.0.1</domain>
    </domain-config>
</network-security-config>
```

##### 2.2 修改AndroidManifest.xml
在`<application>`标签中添加：
```xml
android:networkSecurityConfig="@xml/network_security_config"
```

##### 2.3 修改app/build.gradle
在`buildTypes`的`release`块中添加：
```gradle
resValue "bool", "usesCleartextTraffic", "true"
```

##### 2.4 修改fetchModels函数
添加了对"自定义"提供商的支持，会调用`/models`接口获取模型列表：
```typescript
// 自定义提供商，尝试调用 /models 接口获取模型列表
const response = await fetch(`${baseUrl}/models`, {
  headers: {
    'Authorization': `Bearer ${apiKey}`,
  },
});
```

##### 2.5 添加自动获取模型列表的逻辑
添加了一个useEffect，当用户选择"自定义"提供商，并且输入了Base URL和API密钥后，延迟1秒后自动获取模型列表。

##### 2.6 添加超时控制
在AI服务代码中添加了5分钟超时控制，避免一直等待的问题：
```typescript
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 5 * 60 * 1000);
```

### 最终配置
- **Base URL**: `http://127.0.0.1:8080/v1`
- **API密钥**: `sk`（或任意字符串）
- **模型名称**: `ModelScope/MNN/Qwen3.5-2B-MNN`

### 测试结果
- ✅ 测试连接成功
- ✅ 获取模型列表成功
- ✅ AI解析功能正常工作

### 关键经验
1. **React Native Release版本默认禁止HTTP明文流量**，需要手动配置允许
2. **本地大模型软件MNN chat使用HTTP协议**，需要配置网络安全
3. **本地模型推理可能需要较长时间**（1-2分钟），需要设置合理的超时时间
4. **自动获取模型列表可以提升用户体验**，避免手动输入模型名称

## 模型列表显示UI优化（2026-04-18 10:00-10:21）

### 问题背景
用户反馈：获取大模型列表后，只显示"获取到X个模型"，然后默认选择第一个模型，没有显示模型列表供用户选择。

### 需求
1. 获取模型列表后，显示所有模型供用户选择
2. 不要只显示获取到多少个模型，然后默认选择第一个
3. 需要显示模型列表，让用户可以选择
4. 用户可以输入关键字搜索模型

### 解决方案

#### 1. 添加模型列表显示UI
在SettingsScreen.tsx中添加了模型列表的显示UI：
```typescript
{/* 模型列表 */}
{modelList.length > 0 && (
  <View style={styles.modelListContainer}>
    <Text style={styles.modelListTitle}>获取到 {modelList.length} 个模型，请选择：</Text>
    <ScrollView style={styles.modelList} nestedScrollEnabled>
      {modelList.map((modelName, index) => (
        <TouchableOpacity
          key={index}
          style={[
            styles.modelItem,
            model === modelName && styles.modelItemActive,
          ]}
          onPress={() => setModel(modelName)}
        >
          <Text
            style={[
              styles.modelItemText,
              model === modelName && styles.modelItemTextActive,
            ]}
          >
            {modelName}
          </Text>
        </TouchableOpacity>
      ))}
    </ScrollView>
  </View>
)}
```

#### 2. 添加样式定义
```typescript
modelListContainer: {
  marginTop: 12,
  marginBottom: 8,
},
modelList: {
  marginTop: 8,
  backgroundColor: '#fff',
  borderRadius: 8,
  padding: 8,
  borderWidth: 1,
  borderColor: '#ddd',
  maxHeight: 200,
},
modelListTitle: {
  fontSize: 12,
  color: '#666',
  marginBottom: 8,
},
modelItem: {
  padding: 8,
  borderRadius: 4,
  marginBottom: 4,
  backgroundColor: '#f5f5f5',
},
modelItemActive: {
  backgroundColor: '#8B4513',
},
modelItemText: {
  fontSize: 14,
  color: '#666',
},
modelItemTextActive: {
  color: '#fff',
  fontWeight: 'bold',
},
```

#### 3. 修改fetchModels函数
不再自动选择第一个模型，而是让用户从列表中选择：
```typescript
setModelList(models);
// 不自动选择第一个模型，让用户从列表中选择
Alert.alert('成功', `获取到 ${models.length} 个模型，请从列表中选择`);
```

#### 4. 修复样式定义的缩进问题
修复了`modelListContainer`样式定义的缩进问题，确保样式正确应用。

### 测试结果
- ✅ 获取模型列表后，显示所有模型供用户选择
- ✅ 用户可以点击选择模型
- ✅ 选中的模型会高亮显示（棕色背景）
- ✅ 不再自动选择第一个模型

### 关键经验
1. **UI显示条件**：使用`{modelList.length > 0 && (...)}`来条件渲染模型列表
2. **样式定义**：确保样式定义的缩进正确，避免样式解析错误
3. **用户交互**：提供清晰的视觉反馈，让用户知道哪个模型被选中
4. **ScrollView嵌套**：使用`nestedScrollEnabled`属性，确保在ScrollView中可以正常滚动

// 根据六爻计算卦名
const calculateGuaName = (yaoArr: YaoType[]): { guaName: string; bianGuaName: string } => {
  // 将六爻转换为上下卦
  const xiaYao = yaoArr.slice(0, 3); // 初爻、二爻、三爻
  const shangYao = yaoArr.slice(3, 6); // 四爻、五爻、上爻
  
  // 计算下卦和上卦
  const xiaGua = yaoToGua(xiaYao);
  const shangGua = yaoToGua(shangYao);
  
  // 卦名
  const guaName = GUA64_NAME[shangGua + xiaGua] || shangGua + xiaGua;
  
  // 计算变卦（如果有动爻）
  let bianGuaName = '';
  const dongYaoIndex = yaoArr.findIndex(y => y === '老阴' || y === '老阳');
  
  if (dongYaoIndex !== -1) {
    // 有动爻，计算变卦
    const bianYaoArr = yaoArr.map((y, i) => {
      if (y === '老阴') return '阳';
      if (y === '老阳') return '阴';
      return y;
    });
    
    const bianXiaYao = bianYaoArr.slice(0, 3);
    const bianShangYao = bianYaoArr.slice(3, 6);
    const bianXiaGua = yaoToGua(bianXiaYao);
    const bianShangGua = yaoToGua(bianShangYao);
    bianGuaName = GUA64_NAME[bianShangGua + bianXiaGua] || bianShangGua + bianXiaGua;
  }
  
  return { guaName, bianGuaName };
};

// 爻转卦
const yaoToGua = (yaoArr: YaoType[]): string => {
  const yao = yaoArr.map(y => {
    if (y === '阳' || y === '老阳') return '阳';
    return '阴';
  });
  
  const [y1, y2, y3] = yao;
  
  if (y1 === '阳' && y2 === '阳' && y3 === '阳') return '乾';
  if (y1 === '阴' && y2 === '阴' && y3 === '阴') return '坤';
  if (y1 === '阳' && y2 === '阴' && y3 === '阴') return '震';
  if (y1 === '阴' && y2 === '阳' && y3 === '阳') return '巽';
  if (y1 === '阴' && y2 === '阳' && y3 === '阴') return '坎';
  if (y1 === '阳' && y2 === '阴' && y3 === '阳') return '离';
  if (y1 === '阳' && y2 === '阳' && y3 === '阴') return '艮';
  if (y1 === '阴' && y2 === '阴' && y3 === '阳') return '兑';
  
  return '坤';
};

// 根据六爻计算卦名
const calculateGuaName = (yaoArr: YaoType[]): { guaName: string; bianGuaName: string } => {
  // 将六爻转换为上下卦
  const xiaYao = yaoArr.slice(0, 3); // 初爻、二爻、三爻
  const shangYao = yaoArr.slice(3, 6); // 四爻、五爻、上爻
  
  // 计算下卦和上卦
  const xiaGua = yaoToGua(xiaYao);
  const shangGua = yaoToGua(shangYao);
  
  // 卦名
  const guaName = GUA64_NAME[shangGua + xiaGua] || shangGua + xiaGua;
  
  // 计算变卦（如果有动爻）
  let bianGuaName = '';
  const dongYaoIndex = yaoArr.findIndex(y => y === '老阴' || y === '老阳');
  
  if (dongYaoIndex !== -1) {
    // 有动爻，计算变卦
    const bianYaoArr = yaoArr.map((y, i) => {
      if (y === '老阴') return '阳';
      if (y === '老阳') return '阴';
      return y;
    });
    
    const bianXiaYao = bianYaoArr.slice(0, 3);
    const bianShangYao = bianYaoArr.slice(3, 6);
    const bianXiaGua = yaoToGua(bianXiaYao);
    const bianShangGua = yaoToGua(bianShangYao);
    bianGuaName = GUA64_NAME[bianShangGua + bianXiaGua] || bianShangGua + bianXiaGua;
  }
  
  return { guaName, bianGuaName };
};

// 爻转卦
const yaoToGua = (yaoArr: YaoType[]): string => {
  const yao = yaoArr.map(y => {
    if (y === '阳' || y === '老阳') return '阳';
    return '阴';
  });
  
  const [y1, y2, y3] = yao;
  
  if (y1 === '阳' && y2 === '阳' && y3 === '阳') return '乾';
  if (y1 === '阴' && y2 === '阴' && y3 === '阴') return '坤';
  if (y1 === '阳' && y2 === '阴' && y3 === '阴') return '震';
  if (y1 === '阴' && y2 === '阳' && y3 === '阳') return '巽';
  if (y1 === '阴' && y2 === '阳' && y3 === '阴') return '坎';
  if (y1 === '阳' && y2 === '阴' && y3 === '阳') return '离';
  if (y1 === '阳' && y2 === '阳' && y3 === '阴') return '艮';
  if (y1 === '阴' && y2 === '阴' && y3 === '阳') return '兑';
  
  return '坤';
};
