# 2026-04-13 (Mon) - Divination App Release Build

## Task
Produce a release APK/AAB for the `divination-app` React Native project (version 0.72.7).

## Challenges & Resolutions
- **Maven instability**: 501 errors from Maven Central → configured Aliyun mirrors; pinned AGP 7.4.2 + Gradle 7.6; relied on local Gradle plugin from node_modules.
- **Metro bundling issues**: Initial missing `@react-native/metro-config`, old `metro-config` → installed `metro-config@0.76.8` and `@react-native/metro-config`; created expected output dirs manually to avoid ENOENT; bundled successfully.
- **Kotlin type error in react-native-screens**: `Canvas?` instead of `Canvas` → patched `ScreenStack.kt` in node_modules to use safe call.
- **Package rename**: `ScreensPackage` renamed to `RNScreensPackage` in react-native-screens v3 → updated `MainApplication.kt` imports and list accordingly.
- **Build tools mismatch**: Default build tools 33.0.2 → set `buildToolsVersion = "34.0.0"` to match compileSdk 34.
- **Hermes**: Removed explicit `enableHermes` (RN 0.72+ enables by default).
- **SDK version scope**: Moved `compileSdkVersion`, `minSdkVersion`, etc. to root `build.gradle` `ext` block to propagate to subprojects.
